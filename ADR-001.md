# ADR-001: TypeScript over Python

**Status:** Accepted  
**Date:** 2026-01-04

## Context

The original CHAMBER prototype was written in Python using LangGraph and Google Gemini. As the project scope grew to include a full tool execution pipeline, provider abstraction, and a strict permission model, the Python implementation showed structural limitations — primarily around type safety and the overhead of maintaining a dynamic tool registry without compile-time guarantees.

A rewrite was proposed to address these concerns before the codebase grew further.

## Decision

Rewrite CHAMBER from scratch in TypeScript (strict mode, ESNext modules) running on Node.js via `tsx`. The Anthropic and OpenAI SDKs both have first-class TypeScript support, which eliminates adapter boilerplate. Strict mode catches entire classes of provider-contract bugs at compile time rather than at 2am during a demo.

## Alternatives Considered

| Option | Rationale | Outcome |
| --- | --- | --- |
| Keep Python, add type hints | Incremental. Avoids a rewrite. | Rejected — `mypy` strict on a LangGraph codebase is its own form of suffering. |
| Go | Excellent performance, strong typing, single binary. | Rejected — the team's Go knowledge is theoretical at best and inspirational at worst. |
| Rust | Zero-cost abstractions, memory safety, bragging rights. | Rejected — the time to first working LLM call was estimated at one geological epoch. |
| Staying in bed | Zero technical debt. No architectural decisions to make. Compiles instantly. | Rejected — due to ambition. Revisited periodically. |

## Consequences

- The codebase is fully type-checked via `tsc --noEmit` before any merge.
- Provider contracts (`LLMProvider`, `AgentMessage`) are enforced at compile time.
- New contributors familiar with TypeScript can onboard without understanding the prior Python architecture.
- The original author can no longer read the old Python code without a faint sense of nostalgia for a simpler time. This is not a consequence of TypeScript. It is a consequence of having started the project at all.
